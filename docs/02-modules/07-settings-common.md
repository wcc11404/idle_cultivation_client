# 设置与通用提示

## 职责与边界

- 管理设置页中的本地配置（帧率限制、音频音量/静音）。
- 管理设置页中的入口按钮（兑换码、邮箱、商城、排行榜、游戏说明、登出）。
- 管理顶部账号入口弹窗（昵称修改 + 头像修改）。
- 业务失败文案由模块映射，技术失败由网络层兜底处理。

## 关键状态

- 设置主面板与排行榜子面板切换状态。
- 设置主面板滚动：保留纵向滚动能力，但滚轴不显示（视觉隐藏）。
- 顶部账号展示数据来源：`GameManager.account_info`。
- 本地设置文件：`user://settings.cfg`
  - `graphics.fps_limit`
  - `audio.music_volume`
  - `audio.music_muted`

## API 交互

- `POST /auth/change_nickname`
- `POST /auth/change_avatar`
- `GET /game/rank`
- `POST /auth/logout`

> 备注：帧率限制、音频设置、兑换码/邮箱/游戏说明入口不依赖服务端 API（当前为本地设置或占位入口）。

## 功能触发流转

### 1) 打开设置页

1. 显示设置主面板，默认隐藏排行榜面板。
2. 绑定按钮事件（帧率档位、音频、兑换码、邮箱、商城、排行榜、游戏说明、登出、返回）。

### 2) 点击顶部账号区（头像+昵称）

1. 弹出账号编辑弹窗（昵称输入、非法说明、头像网格、提交按钮）。
2. 修改昵称：
   - 先做本地基础校验（空值、长度、空格、全数字）。
   - 调 `auth/change_nickname`。
   - 成功：
   - 输出“昵称修改成功”。
   - 更新 `GameManager.account_info.nickname`。
   - 调 `game_ui.update_account_ui` 同步顶部显示。
   - 失败：按 reason_code 映射原因（敏感词、非法字符等）。
3. 修改头像：
   - 点击头像缩略图切换选中（黄色边框高亮）。
   - 调 `auth/change_avatar`。
   - 成功更新 `GameManager.account_info.avatar_id` 并刷新顶部头像。
4. 点击弹窗外部区域关闭弹窗；弹窗内部点击不关闭。

### 3) 点击排行榜

1. 切换到排行榜子面板。
2. 调 `game/rank`。
3. 成功：
   - 清空旧列表并重建头部 + 条目。
   - 有数据时默认静默，不额外打印“加载成功”日志。
   - 空数据时输出“排行榜暂无数据”。
4. 失败：输出“排行榜加载失败”映射文案。

### 4) 点击返回

1. 从排行榜子面板回到设置主面板。
2. 不触发额外网络请求。

### 5) 帧率限制设置

1. 通过按钮组选择帧率（`30/60/120/144/不限`）。
2. 点击后立即更新 `Engine.max_fps`，并刷新按钮高亮态。
3. 同步写入 `user://settings.cfg` 的 `graphics.fps_limit`。

### 6) 音频设置

1. 音量滑条范围 `0.0~1.0`，调整后即时应用到音频总线（优先 `Music`，不存在则回退 `Master`）。
2. 点击喇叭按钮切换静音/取消静音（`🔇` / `🔊`）。
3. 右侧百分比标签实时显示当前音量。
4. 静音状态与音量值持久化到 `user://settings.cfg`。

### 7) 兑换码/邮箱/商城/游戏说明

1. 兑换码包含输入框 + `兑换` 按钮。
2. 邮箱、商城、游戏说明为占位按钮。
3. 当前点击统一提示“功能暂未开放”。

### 8) 点击登出

1. 调 `auth/logout`（尽力调用）。
2. 无论接口是否失败，都执行本地 token 清理。
3. 切回登录场景，保证用户可重新登录。

## reason_code 文案策略

- 改昵称/改头像：reason_code 精确映射，避免底层错误直接展示。
- 登出：失败只做必要提示，不阻止本地退出。
- 排行榜：有数据成功静默；空数据或失败时提示。

## 失败处理与回退

- 网络/鉴权异常由 NetworkManager 统一处理。
- 排行榜失败不影响昵称与登出操作。

## 测试覆盖点

- 顶部弹窗改昵称成功与非法昵称映射。
- 排行榜成功静默、失败提示。
- 登出本地 token 清理与场景切换。

## 数值显示约定（当前实现）

- 排行榜灵气列：走普通数值口径（两位小数 / 紧凑单位）。
- 排名序号、境界层数、帧率档位、音量百分比属于特例，保持各自业务表达，不强制转统一紧凑单位。

> 备注：顶部弹窗“改头像”自动化用例当前尚未补齐，现阶段主要依赖手工验收（选中高亮、提交成功、顶部头像刷新）。
> 备注：设置页帧率/音频/商城等本地 UI 交互当前以手工验收为主（自动化尚未单独补用例）。

## 典型触发链路（函数级）

以“点击排行榜”为例：

1. `SettingsModule.on_rank_pressed` 切换到排行榜子面板并调 `api.get_rank`。
2. 成功时 `SettingsModule._render_rank_list` 重建列表；有数据成功保持静默。
3. 空列表时输出“排行榜暂无数据”。
4. 失败时按 `reason_code` 输出失败提示，不展示技术细节。
5. 点击返回由 `on_rank_back_pressed` 仅切面板，不触发额外请求。
